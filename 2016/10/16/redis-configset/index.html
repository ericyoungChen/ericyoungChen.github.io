Redis动态修改运行内存

  • redis

场景: 业务正在运行的时候，遇到redis的内存不够，程序报错。此时通过修改配置文件，再重启redis显然不是最好的办法。这个时候，就可以用到redis内建的命令来为正在运行的redis实例添加运行内存。

操作步骤以及相关介绍

一、 CONFIG GET

该命令可用于取得运行中的 Redis 服务器的配置参数(configuration parameters)，接受单个参数作为搜索关键字。如:

127.0.0.1:6380> CONFIG GET max*
1) "maxmemory"
2) "0"
3) "maxmemory-samples"
4) "3"
5) "maxclients"
6) "10000"
7) "maxmemory-policy"
8) "volatile-lru"
127.0.0.1:6380> CONFIG GET maxmemory
1) "maxmemory"
2) "0

二、 CONFIG SET

该命令可以动态地调整Redis的配置，而无需重启。使用如下:

127.0.0.1:6380> CONFIG GET maxmemory
1) "maxmemory"
2) "0“
127.0.0.1:6380> CONFIG SET maxmemory 100000
OK
127.0.0.1:6380> CONFIG GET maxmemory
1) "maxmemory"
2) "100000"

三、 CONFIG REWRITE

该命令对启动 Redis 服务器时所指定的redis.conf文件进行改写： 因为 CONFIG SET 命令可以对服务器的当前配置进行修改， 而修改后的配置可能和redis.conf文件中所描述的配置不一样，CONFIG REWRITE的作用就是通过尽可能少的修改， 将服务器当前所使用的配置记录到redis.conf文件中。使用如下:

127.0.0.1:6380> CONFIG REWRITE
OK

直接检测redis配置文件，可以看到对配置文件修改操作成功:

$> grep maxmemory redis6380.conf  | grep -v ^#
maxmemory 100000
